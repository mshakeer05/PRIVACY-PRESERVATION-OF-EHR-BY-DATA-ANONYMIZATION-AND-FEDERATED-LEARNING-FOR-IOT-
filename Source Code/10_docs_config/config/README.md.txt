Research Manuscript Project: Privacy-Preserving Federated Learning for Smart City Healthcare



 Modules

 1_data_pipeline/`
- Load & preprocess EHR/IoT datasets
- Normalize & encode
- Attach metadata tags

 2_data_anonymization/`
- DIVA implementation
- NDF filtering, clustering, generalization
- Info loss and disclosure metrics

 3_privacy_architecture/`
- AES encryption
- TLS simulation
- Client-server secure comms

 4_federated_learning/`
- Local client training
- Federated Averaging
- Optional secure aggregation

5_integration/`
- MQTT/Kafka orchestration
- Sync FL updates

 6_evaluation/`
- Centralized vs federated model
- Metrics: Accuracy, Info Loss, Runtime

 7_application_simulation/`
- Real-time Dash dashboard
- Vital monitoring, emergency alerts

8_user_interface/`
- Flask backend + React frontend
- Upload, anonymize, monitor UI

9_security_logging/`
- Action/event logging
- Anomaly detection (payload, accuracy)

10_docs_config/`
- Centralized configs (YAML/JSON)
- Developer documentation



 Configuration Files

| File                | Purpose                          |
|---------------------|----------------------------------|
| `diva_config.yaml`  | DIVA anonymization parameters    |
| `fl_config.yaml`    | Federated learning tuning        |
| `iot_sim_config.yaml` | IoT sensor simulator settings  |
| `runtime_config.json` | Runtime toggles (dev/prod, UI) |



 How to Run the System

1. Install Backend:
bash
cd 8_user_interface/flask_app
pip install -r requirements.txt
python app.py
